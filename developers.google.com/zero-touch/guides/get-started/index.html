Get started

Before you can call zero-touch enrollment reseller API methods, you need to set up access. This document explains how you can start using the zero-touch enrollment API.

Before you start

Before you use the API, confirm that you're ready. Prerequisites are:

  1. You're a device reseller.
  2. You've onboarded in zero-touch enrollment.

If you're a customer and not a device reseller, visit Android Enterprise Help to learn more about zero-touch enrollment. If you haven't onboarded yet, follow the steps in Get started in the Reseller portal guide.

Step 1. Get a service account

You call APIs using a service account that represents your organization. Follow the steps in Authorization to set up and authorize an API service account.

Step 2: Install a Google API client library

You can use a Google API client library to make coding against the reseller API easier. The client libraries also make it simpler to set up authorization and authentication. Client libraries are available for many popular languages, including Java and .NET. For a guide to help you install a library for your language, see Google API Client Libraries.

The shell command below shows how you might install the client library for Python using pip. For more information, see Python Installation.

pip install --upgrade google-api-python-client

If you can't find a Google API client library for your language, you can use your language's networking libraries to make the HTTPS requests.

Samples in this site

To help you try the examples in the documentation, use the authorization snippet below. The examples in this site share common variable names, so you can copy and paste each snippet after the authorization setup lines.

#!/usr/bin/env python
from oauth2client.service_account import ServiceAccountCredentials
from httplib2 import Http
from apiclient.discovery import build

# Replace these with your partner ID and values from your test data set
MY_PARTNER_ID = '123456789012345678'
TARGET_CUSTOMER = '1234567890123456789'
TARGET_DEVICE_ID = '123456789012345'

scopes = ['https://www.googleapis.com/auth/androidworkprovisioning']

credentials = ServiceAccountCredentials.from_json_keyfile_name('zt-prod-key.json', scopes=scopes)

http_auth = credentials.authorize(Http())
provisioning = build('androiddeviceprovisioning', 'v1', http=http_auth)

# Above is the standard API call setup used for all examples in this site
# -------------------------------------------------------------------

# Call an API method with the authorization
results = provisioning.partners().customers().list(partnerId=MY_PARTNER_ID).execute()

# Print out the response body containing JSON
print results

To use this Python setup example to try the API snippets, follow the steps below.

  1. Copy the example script above into a text file and save as tryout_api.py.
  2. Set MY_PARTNER_ID to your reseller partner ID. See Partner ID below for help.
  3. For some methods, you might need to replace the constants with IDs and values from your test data set.
  4. Change the string zt-prod-key.json, to the name and path of your JSON key file.
  5. Replace the line that begins results = with the API call from the documentation.

Use your operating system's help to run the script in the file. On UNIX and Mac computers, run the command below in your terminal:

python tryout_api.py

Partner ID

API calls typically need your reseller partner ID as an argument. To find your partner ID from the zero-touch enrollment portal, follow the steps below:

  1. Open the portal. You might need to sign in.
  2. Click account_box Service accounts.
  3. Copy your partner ID number from the Your reseller ID line.

JSON key file

In the example above, we store the service account key file in the same directory as our script. You should download the JSON key file and securely store it in a location that only your app can access.

Important: Don't store the JSON key file in a publicly accessible location. If you share the source code to your app—for example, on GitHub—store the JSON key file outside of your source tree to avoid inadvertently sharing your client credentials.

Printing API responses

To make it easier to inspect responses when trying out the API, format the JSON response data. The snippet below shows how you might do this in Python using the JSON module:

from json import dumps

# ...

results = provisioning.partners().devices().claimAsync(partnerId=MY_PARTNER_ID,
 body={'claims':new_claims}).execute()
# Print formatted JSON response
print dumps(results, indent=4, sort_keys=True)
